# Mamba-3 checkpoint availability

**Status as of 2026-04-14:** no official Mamba-3 SISO checkpoints are
publicly available.

## Official channels

| Source | Mamba-3 status |
|--------|-----------------|
| [`state-spaces/mamba`](https://github.com/state-spaces/mamba) GitHub | Source-only (`modules/mamba3.py`). README lists pretrained weights for Mamba-1 (130m/370m/790m/1.4b/2.8b) and Mamba-2 (2.7b) but **no Mamba-3 entries**. |
| `state-spaces/*` on HuggingFace (Tri Dao, Albert Gu) | Mamba-1 and Mamba-2 variants only. No Mamba-3 upload from any paper author. |
| [state-spaces/mamba issue #860](https://github.com/state-spaces/mamba/issues/860) | "Will Mamba 3 be released on Hugging Face?" — open, no maintainer response. |
| [arXiv 2603.15569](https://arxiv.org/abs/2603.15569) | Paper describes 1.5B-scale experiments. No checkpoint link in the paper or in supplementary materials on ICLR OpenReview. |
| [Together AI — Mamba-3 blog](https://www.together.ai/blog/mamba-3) | Discusses kernel releases (Triton / TileLang / CuTe DSL). No weights released. |

## Implications for mamba-rs

- **Inference tests on real LLMs** use Mamba-1 checkpoints only
  (mamba-130m/370m/1.4b/2.8b-hf). Mamba-3 is exercised against
  **synthetic weights** generated by `Mamba3Weights::init`.
- **Benchmarks**: `tests/bench_bf16_vs_f32.rs` has a `bench_m3_bf16_vs_f32_synthetic`
  path; `tests/m3_gpu_benchmark.rs` runs synthetic M3 end-to-end.
- **Correctness**: `tests/m3_test_correctness.rs` and
  `tests/m3_bf16_parity.rs` exercise synthetic weights against
  finite-difference gradients and f32/bf16 parity.

## Monitoring upstream

If/when a Mamba-3 SISO checkpoint lands under `state-spaces/mamba-3-*-hf`
on HuggingFace, adding it to the parametric parity test is a one-liner —
`candidates` array in `tests/gpu_bf16_parity.rs::test_gpu_lm_bf16_matches_f32_all_cached_models`.
The loader infrastructure (`src/hf/`) should "just work" for any M3
checkpoint following the `state-spaces/mamba-*-hf` naming convention,
but the HF config parser and weight key remapper may need small tweaks
(the current code is tuned for Mamba-1's HF config schema).
